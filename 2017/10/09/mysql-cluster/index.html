注意

s74的数据库结构数据保存在以下目录中，因为 在 /etc/my.conf中没有配置相关数据文件目录
/usr/local/mysql-cluster-gpl-7.4.7-linux-glibc2.5-x86_64/

总结

  1. 集群是不需要安装mysql server的
  2. 数据节点和sql节点都需要my.conf的配置文件
  3. 管理节点需要config.ini来告诉它有多少分片、sql节点和数据节点在哪等
  4. 集群没启动起来之前，不要随便修改表结构和插入数据
  5. 由管理节点来协调数据节点、SQL节点之间的调用
  6. 集群最重要的部分就是管理节点，管理节点挂掉了整个集群就不可用了
  7. mysql集群的启动顺序是很有讲究的，要先启动管理节点，然后再启动数据节点、最后再启动SQL节点

解决方法
参考MySQL Cluster - Table is full [Error 1114] while still creating tables

某一个表卡死

问题描述

Got temporary error 233 ‘Out of operation records in transaction coordinator (increase MaxNoOfConcurrentOperations)’ from NDBCLUSTER

问题描述
在用navicat还原数据库时，表创建完，恢复到数据时报这个错误

解决方法：

节点无法启动的问题

  1. 随着数据的增多，节点的启动时间会变成，一般数据节点启动之后，再启动SQL节点。启动SQL节点的时间一般需要两三分钟以上
  2. 只要启动顺序没问题就行了
